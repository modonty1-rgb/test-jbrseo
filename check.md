# القسم الأول — الأخطاء الحرجة 🔴

## المشكلة ١ — عنوان الصفحة: "test \| JBRSEO"

### ما وجدناه
```
<title>test | JBRSEO</title>
```
كلمة **"test"** ظاهرة كعنوان للصفحة في المتصفح وفي نتائج جوجل.

### التأثير
- العميل الذي يفتح التبويب يرى "test" → يظن الموقع غير مكتمل ويغلق
- لو ظهرت الصفحة في جوجل، تظهر بعنوان "test" → لا أحد يضغط
- يبطل أي جهد SEO

### الحل المطلوب
```tsx
// في layout.tsx أو في صفحة /sa مباشرة
export const metadata: Metadata = {
  title: 'مدونتي — محتوى يجلب عملاء من السعودية بدون إعلانات',
  // أو حسب الجمهور المستهدف
}
```

---

## المشكلة ٢ — بق `{n} {c}` في كل بطاقات الأسعار

### ما وجدناه
```
يُدفع {n} {c} · تحصل على ١٨ شهراً
```
هذا النص يظهر **٣ مرات** — في كل باقة مدفوعة (الانطلاقة، الزخم، الريادة).

### التأثير
- العميل الجاهز للشراء يرى placeholder غير محلول → يشك في الموقع كله
- يوحي أن الموقع غير جاهز للإنتاج
- أي إعلان يوصله لهذه الصفحة = خسارة مباشرة

### الحل المطلوب
```tsx
// مثال: إذا المتغيرات billing period وcurrency
const billingLabel = billing === 'annual' ? 'سنوياً' : 'شهرياً'
const currency = 'ر.س'

// النص الصح:
`يُدفع ${billingLabel} بـ ${currency} · تحصل على ١٨ شهراً`

// أو بالعربي الكامل:
`يُدفع سنوياً · تحصل على ١٨ شهراً`
```

## المشكلة ٣ — OG Tags مكسورة (property = undefined)

### ما وجدناه
```json
og: [
  "undefined: test",
  "undefined: JBRSEO – وكالة سيو...",
  "undefined: https://www.jbrseo.com/sa"
]
```
الـ `property` attribute في جميع OG tags يُعاد كـ `undefined` — يعني الصفحة لا تملك OG tags تعمل فعلياً.

### التأثير
- لما تشارك الرابط على واتساب أو تويتر أو لينكدإن — لا يظهر أي preview
- حملات الـ Social Media Ads تفقد الـ thumbnail والعنوان الجذاب
- يقلل click-through rate بشكل كبير

### الحل المطلوب
```tsx
// في /sa/page.tsx أو layout.tsx
export const metadata: Metadata = {
  title: 'مدونتي — محتوى يجلب عملاء من السعودية',
  description: 'منصة المحتوى العربي — مقالات تتصدر جوجل وتجلب عملاء بدون إعلانات. جرّب مجاناً.',
  openGraph: {
    title: 'مدونتي — ابدأ بمقال مجاني',
    description: 'محتوى يجلب عملاء بدون إعلانات. +١٢٠ نشاط تجاري في السعودية.',
    url: 'https://www.jbrseo.com/sa',
    type: 'website',
    locale: 'ar_SA',
    images: [{ url: 'https://...og-image.png', width: 1200, height: 630 }],
  },
}
```

## المشكلة ٤ — الـ H1 يخاطب جمهوراً خاطئاً

### ما وجدناه
```
H1: "اتوسع في سوق المملكة بفرع رقمي لشركتك"
```
والـ subtext يقول: *"لا تنتظر السائح السعودي يبحث عنك... يجلب لك العملاء لقلب مصر"*

### التأثير
- الصفحة `/sa` يُفترض أنها تستهدف الشركات **السعودية** من إعلانات سعودية
- لكن الرسالة كاملة مكتوبة للشركة **المصرية** التي تبي تدخل السعودية
- العميل السعودي الجاهز للدفع يقرأ هذا ويقول "هذا مو عني" ويغلق

### قرار مطلوب منك أولاً
**من هو الجمهور الأساسي لهذه الصفحة؟**

| الخيار | الجمهور | الرسالة |
|---|---|---|
| أ | شركة سعودية تبي تنمو داخل السعودية | "اجلب عملاء سعوديين من جوجل — بدون إعلانات" |
| ب | شركة مصرية تبي تدخل السوق السعودية | "افتح فرعاً رقمياً في السعودية — بدون مكتب" |
| ج | الاثنان معاً | صفحتان منفصلتان: `/sa` و `/eg` |

### الحل المقترح (إذا الجمهور سعودي)
```
H1: "اجلب عملاء جدد كل شهر — بدون إعلانات"
Subheadline: "محتوى يتصدر جوجل السعودية ويجيب على أسئلة عملائك قبل ما ينافسك أحد"
```

## المشكلة ٦ — صور الشهادات الثلاث لا تُحمَّل

### ما وجدناه
```json
{ "alt": "أحمد العتيبي", "src": "" }
{ "alt": "منى شريف",    "src": "" }
{ "alt": "فيصل الدوسري", "src": "" }
```
كل صور الشهادات تُرجع `src: ""` — الصور ما تتحمل فعلياً من الـ DOM.

### التأثير
- قسم الشهادات يظهر بصور مكسورة أو فارغة
- أسوأ ما يمكن لصفحة تطلب ثقة مالية من العميل

### الحل المطلوب
```tsx
// خيار ١: استخدم صور حقيقية من Cloudinary
<Image
  src="https://res.cloudinary.com/.../ahmed-atibi.jpg"
  alt="أحمد العتيبي"
  width={64}
  height={64}
  className="rounded-full"
/>

// خيار ٢: fallback avatar إذا الصورة ما توفرت
const [imgError, setImgError] = useState(false)
<Image
  src={imgError ? '/avatars/default.svg' : testimonial.image}
  onError={() => setImgError(true)}
  ...
/>

// خيار ٣ مؤقت: استخدم الأحرف الأولى كـ avatar
<div className="w-16 h-16 rounded-full bg-primary flex items-center justify-center">
  <span className="text-xl font-bold text-white">أ</span>
</div>
```

## المشكلة ٧ — تضارب اللهجة: مصري وسعودي في نفس الصفحة

### ما وجدناه في الـ FAQ
```
❌ مصري: "إيه اللي هستلمه بالظبط في باقة التأسيس؟"
❌ مصري: "لو قررت أوقف الاشتراك — هيحصل إيه في المحتوى؟"
❌ مصري: "في عقد أو التزام طويل هيقيدني؟"
❌ مصري: "كيف تكتبون المقالات؟ لازم أشارك أو أتعب؟"
```

وفي أماكن أخرى لهجة خليجية:
```
✓ "بدون عقود — ألغِ في أي وقت"
✓ "دورك الموافقة فقط"
```

### التأثير
- السعودي يقرأ "إيه اللي هستلمه" ويحس الخدمة مو موجهة له
- تكسر الـ tone of voice وتضعف الثقة

### الحل المطلوب — أعِد كتابة FAQ بلهجة موحدة

إذا الجمهور سعودي:
```
❌ "إيه اللي هستلمه؟"     → ✅ "وش اللي بآخذه بالضبط؟"
❌ "هيحصل إيه؟"           → ✅ "وش يصير بالمحتوى؟"
❌ "في عقد هيقيدني؟"      → ✅ "في عقد يلزمني؟"
❌ "لازم أشارك؟"          → ✅ "لازم أشارك؟" (هذي مشتركة — تمام)
```

## المشكلة ٨ — ترتيب الأقسام عكس منطق الشراء

### الترتيب الحالي
```
١. Hero
٢. كيف نعمل (٣ خطوات)
٣. Calculator ROI ← ثقيل جداً مبكراً
٤. النتائج (أرقام)
٥. الشهادات
٦. الأسعار
٧. FAQ
```

### المشكلة
العميل يوصل الـ Calculator الثقيل في خطوته الثانية — قبل ما يقتنع بالمنتج أصلاً.
والشهادات تأتي **بعد** الأسعار — والعميل محتاج يقتنع **قبل** ما يشوف السعر.

### الترتيب الصح (مبني على Conversion Rate Optimization)
```
١. Hero → رسالة واضحة + CTA رئيسي
٢. Social Proof Bar → شعارات الشركات (ثقة فورية)
٣. النتائج → أرقام قابلة للقياس (+٣٧٠٠٪، ٩٠ يوم)
٤. الشهادات → قصص حقيقية قبل السعر
٥. كيف نعمل → ٣ خطوات بسيطة
٦. الأسعار → بعد الاقتناع الكامل
٧. Calculator ROI → بعد الأسعار كـ justification
٨. FAQ
٩. Final CTA
```

## المشكلة ٩ — ٨+ CTAs متضاربة في صفحة واحدة

### ما وجدناه (من تحليل الأزرار)
```
- شوف الأسعار (Navbar)
- شوف الأسعار والخطة المناسبة ← (Hero)
- تحدث معنا أولاً (Hero)
- شوف الأسعار والخطة المناسبة ← (بعد Calculator)
- شوف جزء مما ستحصل عليه مع مدونتي ← (بعد Calculator)
- ابدأ مجاناً — ما نطلب منك شيء (باقة مجانية)
- جرّب ١٤ يوم — ادفع بس لو اقتنعت (باقة انطلاقة)
- ابدأ الحين — احنا نتولى الباقي (باقة زخم)
- احجز جلستك المجانية (باقة ريادة)
- تواصل معنا على واتساب الآن
- ابدأ مجاناً — ما نطلب منك شيء (Final CTA)
- كلّمنا على واتساب (Final CTA)
```

**١٢ زر بـ ٧ رسائل مختلفة = شلل القرار**

## المشكلة ١٠ — Scarcity Counter ثابت وغير حقيقي

### ما وجدناه
```
"تبقى ٦٠ مقعداً فقط من ١٥٠"
```
الرقم ثابت في كل زيارة — لا يتغير، لا countdown، لا تحديث.

### التأثير
- العميل الذكي يلاحظ الرقم ثابت → يفقد الثقة بكل الصفحة
- Fake scarcity أسوأ من عدم وجود scarcity

### الحل: إما حقيقي أو احذفه

**الخيار أ — حقيقي (الأفضل):**
```tsx
// احسب من database: كم سجّل فعلياً
const remaining = MAX_SEATS - registeredCount
// وحدّث كل ساعة أو عند كل تسجيل
```

**الخيار ب — احذفه واستبدله بـ urgency حقيقي:**
```
"سعر التأسيس ينتهي [تاريخ محدد]"
// مع countdown timer حقيقي
```

**الخيار ج — احذفه تماماً:**
أفضل من scarcity كاذب.

### الحالة الحالية (بعد التنفيذ — ٢٠٢٦)

تمت إزالة العداد الرقمي الثابت من الواجهة. الشريط العلوي و**Final CTA** يعرضان **نصاً واحداً** يُدار من إعدادات المحتوى/الهيدر في الـ Admin — لا يُطلب «إصلاح تقني» إضافي هنا؛ يبقى فقط **مراجعة صياغة النص** بحيث تكون صادقة. التفاصيل: [check-technical.md](./check-technical.md)، [check-content.md](./check-content.md).

## المشكلة ١٣ — ١٨٢ عنصر نص بخط أصغر من ١٤px

### ما وجدناه
```javascript
smallTextElements: 182  // عناصر p/span/small بـ font-size < 14px
```

### التأثير
- قراءة صعبة على الموبايل
- يفشل معيار WCAG AA للـ accessibility
- يضر بتجربة المستخدم الكبير في السن

### الحل
```css
/* في globals.css أو Tailwind config */
/* لا تستخدم text-xs (12px) للنصوص المهمة */

/* الحد الأدنى المقبول */
p, span { font-size: 14px; /* text-sm */ }
/* للـ helper text فقط */
.caption { font-size: 12px; }
```

```tsx
// في tailwind.config.ts
theme: {
  extend: {
    fontSize: {
      xs: ['0.75rem', { lineHeight: '1.5' }],  // 12px — للـ captions فقط
      sm: ['0.875rem', { lineHeight: '1.6' }], // 14px — الحد الأدنى للنصوص
    }
  }
}
```

# ملاحظات ختامية — للمطور

## الأسرع تأثيراً على الحملة الإعلانية

لو عندك ٤ ساعات فقط قبل الإطلاق، ركّز على:

1. **`{n} {c}` bug** ← أوقفه أولاً — كل دقيقة يظهر فيها تخسر فيها عميل
2. **عنوان الصفحة** ← من "test" إلى اسم حقيقي — دقيقتان فقط
3. **H1** ← قرر: السعودي أم المصري؟ واكتب رسالة واحدة واضحة
4. **OG Tags** ← لأن إعلانات السوشال تعتمد عليها للـ preview

هذي الأربعة تأثيرها مباشر على كل ريال تصرفه في الإعلانات.
